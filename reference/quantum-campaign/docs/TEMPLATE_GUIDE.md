# 템플릿 설정 가이드

템플릿 관리 페이지(`/settings/templates`)에서 캠페인 템플릿을 생성/수정/삭제할 수 있습니다.
이 문서는 각 템플릿 필드가 superap.io 캠페인 등록 폼의 어떤 필드에 매핑되는지 설명합니다.

---

## 필드 매핑 테이블

| 템플릿 필드 | superap.io 폼 필드 | 설명 |
|---|---|---|
| **캠페인 이름** (type_name) | - | 엑셀 업로드 시 `캠페인타입` 열에 사용하는 이름. superap.io에 직접 전달되지 않음 |
| **superap.io 캠페인 타입** (campaign_type_selection) | 캠페인 타입 라디오 버튼 | `플레이스 퀴즈`, `검색 후 정답 입력` 등 |
| **모듈** (modules) | - | 등록 전 자동 실행. `landmark`: 명소 추출, `steps`: 걸음수 계산 |
| **전환 인식 방식** | 전환 인식 기준 | 걸음수: 숫자 입력 / 텍스트: 문자열 입력 |
| **참여 방법 설명** (description_template) | 참여 방법 설명 textarea | 변수 자동 치환됨 |
| **정답 힌트** (hint_text) | 정답 맞추기 힌트 | 그대로 입력됨 |
| **링크** (links) | 링크 입력 필드들 | 최대 3개. 해시태그 포함된 URL |
| **해시태그** (hashtag) | 링크 URL 뒤에 자동 추가 | `#cpc_detail_place` 등 |
| **가로 이미지 URL** (image_url_200x600) | 가로 소재 이미지 (200x600) | 이미지 업로드 URL |
| **세로 이미지 URL** (image_url_720x780) | 세로 소재 이미지 (720x780) | 이미지 업로드 URL |

## superap.io 캠페인 이름 자동 생성 규칙

superap.io의 "캠페인 이름" 필드는 템플릿에서 설정하지 않습니다.
상호명 기반으로 자동 생성됩니다:

| superap 타입 | 규칙 | 예시 (상호명: 일류곱창) |
|---|---|---|
| traffic (트래픽/명소) | `{앞2글자} 퀴즈 맞추기` | 일류 퀴즈 맞추기 |
| save (저장하기) | `{앞2글자} 저장 퀴즈 맞추기` | 일류 저장 퀴즈 맞추기 |

공백을 제외한 상호명에서 앞 2글자를 추출합니다.

---

## 변수 치환 규칙

참여 방법 설명, 전환 인식 텍스트에서 사용 가능한 변수:

| 변수 | 치환 값 | 출처 |
|---|---|---|
| `&상호명&` | 마스킹된 상호명 (참여 방법 설명에서만) | 마스킹 로직 |
| `&명소명&` | 명소 이름 | landmark 모듈 |
| `&명소순번&` | N번째 명소 순번 (1부터) | landmark 모듈 |
| `&걸음수&` | 걸음 수 | steps 모듈 |

### 상호명 마스킹 규칙

참여 방법 설명에서 `&상호명&`은 마스킹된 상호명으로 치환됩니다.
공백을 제외하고 2번째, 4번째, 6번째... 글자를 `X`로 교체합니다.

| 원본 | 마스킹 결과 |
|---|---|
| 일류곱창 | 일X곱X |
| 일류곱창 마포공덕본점 | 일X곱X 마X공X본X |
| 박정기법무사사무소 | 박X기X무X사X무X |
| AB | AX |
| A | A |

## 전환 인식 기준 설정

### 걸음수 방식
- **모듈**: `landmark` + `steps` 필수
- **동작**: 명소에서 목적지까지 도보 걸음수를 자동 계산하여 숫자로 입력
- **예시**: `863` (숫자만 입력됨)

### 텍스트 방식
- **모듈**: `landmark` 필수 (steps 불필요)
- **동작**: 설정한 텍스트 템플릿에 변수를 치환하여 입력
- **예시**: `&명소명& ㄱㄱ` → `다산중앙공원 ㄱㄱ`

## 캠페인 타입별 권장 설정

### 트래픽 (기본)
| 항목 | 설정 값 |
|---|---|
| 캠페인 이름 | `트래픽` |
| superap.io 캠페인 타입 | `플레이스 퀴즈` |
| 모듈 | `landmark`, `steps` |
| 전환 인식 방식 | 걸음수 |
| 해시태그 | `#cpc_detail_place` |

### 저장하기
| 항목 | 설정 값 |
|---|---|
| 캠페인 이름 | `저장하기` |
| superap.io 캠페인 타입 | `검색 후 정답 입력` |
| 모듈 | `landmark`, `steps` |
| 전환 인식 방식 | 걸음수 |
| 해시태그 | `#cpc_detail_place` |

### 명소 (텍스트 전환)
| 항목 | 설정 값 |
|---|---|
| 캠페인 이름 | `명소` |
| superap.io 캠페인 타입 | `플레이스 퀴즈` |
| 모듈 | `landmark` (steps 불필요) |
| 전환 인식 방식 | 텍스트: `&명소명&` |
| 정답 힌트 | `&명소순번&번째 명소 이름 맞추기` |
| 참여 방법 설명 | `&명소순번&번째` 변수로 랜덤 명소 순번 표시 |
| 해시태그 | `#cpc_detail_place` |

> **명소 선택 로직**: steps 모듈이 없으면 주변 명소 1~5개 중 **랜덤** 선택. `&명소순번&`에 해당 명소의 순번(1-based)이 자동 치환됨.

## 새 템플릿 생성 체크리스트

1. **캠페인 이름** 입력 (엑셀에서 사용할 타입명, 중복 불가)
2. **캠페인 타입** 셀렉트에서 superap.io의 실제 라디오 버튼과 동일한 옵션 선택
3. **모듈** 선택:
   - 걸음수 필요 → `landmark` + `steps`
   - 명소만 필요 → `landmark`
4. **전환 인식 방식** 선택:
   - 걸음수: steps 모듈 자동 활성화
   - 텍스트: 전환 텍스트 템플릿 입력 (예: `&명소명& ㄱㄱ`)
5. **참여 방법 설명** 작성:
   - `&상호명&`: 마스킹된 상호명
   - `&명소명&`: 명소 이름
   - `&명소순번&`: N번째 명소 순번
   - `&걸음수&`: 걸음 수
   - `{{image|URL}}`: 이미지 삽입
6. **정답 힌트** 입력
7. **링크** 3개 추가 (네이버 로그인, 브릿지, 지도 앱)
8. **해시태그** 선택
9. 저장 후 엑셀 업로드에서 해당 타입명으로 테스트

---

## 주의사항

- 활성 캠페인이 사용 중인 템플릿은 삭제할 수 없습니다
- 캠페인 이름(type_name)을 변경하면 기존 캠페인의 타입과 불일치할 수 있으므로 주의
- superap.io 폼의 라디오 버튼은 자동화 코드에서 `campaign_type_selection` 값으로 선택됩니다
  - 현재 자동화에서 지원하는 타입: `traffic` (플레이스 퀴즈 등), `save` (검색 후 정답 입력)
  - 새 타입 추가 시 `auto_registration.py`의 `type_map`에도 매핑 추가 필요
- 전환 인식 텍스트의 변수(`&명소명&` 등)가 올바른지 확인 — 모듈이 해당 변수를 출력하지 않으면 치환되지 않음
