## 1. 계정 판별 테스트 (Task 1.3)

### 테스트 목적
superap.io가 "어떤 계정의 캠페인인지" 판별하는 기준 파악

### 테스트 결과

| 테스트 | 조건 | 결과 | 비고 |
|--------|------|------|------|
| 테스트 1 | 같은 IP, 다른 일반 브라우저 | ❌ 최근 로그인 계정으로 등록 | 쿠키 공유로 인한 세션 혼선 |
| 테스트 2 | 같은 IP, 시크릿 모드 분리 | ✅ 각각 올바른 계정에 등록 | 쿠키 격리로 정상 작동 |

### 분석 결과

#### 판별 기준
- [x] **쿠키/세션 기반** - 확인됨
- [ ] IP 기반 - 아님
- [ ] Fingerprint - 아님
- [ ] API 토큰 - 아님

#### 상세 분석
1. **테스트 1 (일반 브라우저)**
   - 브라우저 A: 계정A 로그인 → 캠페인 등록
   - 브라우저 B: 계정B 로그인 → 캠페인 등록
   - 결과: 두 캠페인 모두 "최근 로그인 계정"에 등록됨
   - 원인: 일반 브라우저는 시스템 쿠키 저장소를 공유함

2. **테스트 2 (시크릿/컨텍스트 분리)**
   - 컨텍스트 A (계정A): 캠페인 → 계정A에 정상 등록
   - 컨텍스트 B (계정B): 캠페인 → 계정B에 정상 등록
   - 결과: 브라우저 컨텍스트를 분리하면 세션이 격리됨

### 결론
superap.io는 **쿠키/세션 기반**으로 계정을 판별함.
→ **Playwright 브라우저 컨텍스트 분리**로 다중 계정 동시 처리 가능

## 2. 동시 업로드 테스트 (Task 1.4)

### 테스트 목적
한 계정에 여러 캠페인 동시 등록 시 충돌 여부 확인

### 테스트 결과

| 시나리오 | 조건 | 결과 | 권장 처리 방식 |
|----------|------|------|----------------|
| 계정 간 | 다른 계정, 동시 등록 | ✅ 충돌 없음 | 병렬 처리 가능 |
| 계정 내 | 같은 계정, 동시 등록 | ⚠️ 세션 혼선 가능 | 순차 처리 권장 |

### 상세 분석

#### 계정 간 병렬 처리
- 각 계정이 별도 브라우저 컨텍스트를 사용하면 충돌 없음
- 병렬로 캠페인 등록 가능

#### 계정 내 순차 처리
- 같은 계정에서 여러 캠페인을 동시에 등록하면:
  - 폼 데이터 혼선 가능성
  - 서버 측 세션 상태 충돌 위험
- 안전을 위해 계정 내에서는 순차 처리 권장

### 결론
- **계정 간**: 병렬 처리 (컨텍스트 분리)
- **계정 내**: 순차 처리 (큐 방식)
- **충돌 가능성**: 없음 (위 규칙 준수 시)

### 캠페인 처리 전략

```python
# 계정 간 병렬, 계정 내 순차
async def process_all_campaigns(campaigns_by_account):
    await asyncio.gather(
        worker(context_a, campaigns_a),  # 계정A 캠페인들 (내부 순차)
        worker(context_b, campaigns_b),  # 계정B 캠페인들 (내부 순차)
        worker(context_c, campaigns_c),  # 계정C 캠페인들 (내부 순차)
    )

async def worker(context, campaigns):
    """계정 내에서는 순차 처리"""
    for campaign in campaigns:
        await register_campaign(context, campaign)
```

## 4. 설계 결정 요약

| 항목 | 결정 | 이유 |
|------|------|------|

## 5. 성능 예측

### 처리 속도 (예상)
- 캠페인 1건 등록: 약 30~60초 (네이버맵 조회 + superap 등록)
- 계정 3개 × 10캠페인 = 30캠페인
- 병렬 처리 시: 약 5~10분 (계정별 10분 소요, 동시 진행)
- 순차 처리 시: 약 15~30분

### 리소스 사용
- 브라우저 컨텍스트당 메모리: 약 100~200MB
- 계정 3개 동시 처리: 약 300~600MB
- CPU: 병렬 처리 시 코어 수에 따라 분산
