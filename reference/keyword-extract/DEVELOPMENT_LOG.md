> 📅 최종 업데이트: 2026-01-03

## 📋 프로젝트 개요

**목표**: 네이버 플레이스 URL과 메인 키워드를 입력받아, 플레이스 구좌가 노출되는 키워드를 자동으로 생성하는 프로그램

**입력**:
- 네이버 플레이스 URL
- 메인 노출 키워드

**출력**:
- 수십~수백 개의 조합된 키워드 목록
- 각 키워드의 신지도/구지도 타입
- 플레이스 순위 정보

---

## 🗂️ 업종별 URL 구분

| URL 카테고리 | 업종 | 특이사항 |
|:---:|:---|:---|
| `place` | 일반 업종 (꽃집, 학원 등) | 구지도 형태가 대부분 |
| `restaurant` | 맛집/음식점 | 신지도 UI, 필터 풍부 |
| `hospital` | 병의원 | 전문의 정보, 진료과목 필터 |
| `hairshop` | 미용실 | 스타일 칩 (댄디컷, 레이어드 등) |
| `nailshop` | 네일샵 | 스타일 필터 존재 |

## 🗺️ 신지도 vs 구지도 구분

### 신지도 특징
- 도메인: `m.place.naver.com` 연결
- 상단에 동적 필터 칩 존재
- 다중 이미지 갤러리
- 리뷰 스니펫, 별점 강조
- 업종별 전용 UI 존재

### 구지도 특징
- 전화/예약/길찾기 버튼 중심의 단순 리스트
- 필터 칩 없음
- 이미지 갤러리 제한적

### 신지도 → 구지도 전환 트리거
- 키워드에 **"지도"** 추가 시 (예: `홍대 맛집` → `홍대 맛집 지도`)
- 롱테일/특수 키워드 사용 시
- 네이버가 UI 고도화하지 않은 업종

## 📝 업종별 키워드 생성 로직

### 1. 일반 업종 (구지도 - place)

**조합 공식**:
```
[지역] + [업종/품목] + [수식어(선택)]
```

**지역 소스**:
- 행정구역: 마포구, 서대문구, 용산구
- 동네: 연남동, 동교동, 서교동
- 랜드마크: 홍대입구역, 상수역, 합정역
- 일반명: 홍대, 연남, 합정

**업종/품목 소스**:
- 업종: 꽃집, 플라워샵
- 서비스: 꽃배달, 꽃배송
- 품목: 화환, 꽃다발, 생화
- 목적: 기념일 꽃, 생일 꽃

**수식어**:
- 추천, 유명한, 예쁜, 근처, 주변

**플레이스 ❌ 노출 키워드**:
- 정보성 키워드 (예: ~관리법, ~방법, ~가격비교)

---

### 2. 병의원 (hospital)

**조합 공식**:
```
[지역] + [업종 카테고리/진료과목] + [대표키워드]
```

**노출 우선순위**:
1. 업종 카테고리 (진료과)
2. 진료과목 (플레이스 내부)
3. 대표키워드

**추가 조합 규칙**:
- 신지도 키워드 + `병원`, `의원`, `병의원` → 별도 신지도 키워드 생성 가능
- 지역 + (인근/근처/부근/주변/가까운) → 유사 검색결과 (별도 키워드)

**상호명 검색 시 신지도/구지도 분기**:
- 신지도: 풍부한 이미지, 전문의 정보 칩, 리뷰 풍부
- 구지도: 단순 리스트, 전문의 필터 없음

---

### 🎯 키워드 노출 우선순위 (구지도 기준)

> 📅 2026-01-04 추가

**지역 + (키워드)** 조합에서 **키워드의 노출 우선순위**:

| 순위 | 키워드 종류 | 예시 | 비고 |
|:---:|:---|:---|:---|
| 1 | **상호명** | 리더스피부과, 법무법인 영웅 | 가장 높은 우선순위 |
| 2 | **업종 카테고리** | 피부과, 법률사무소, 미용실 | 플레이스 기본 카테고리 |
| 3a | **플레이스 필터** | 영업중, 예약, 주차 | 신지도 전용 |
| 3b | **진료과목** | 정형외과, 피부과, 치주과 | 병의원 전용 |
| 4 | **대표 키워드** | 울쎄라, 보톡스, 강남변호사 | keywordList |

> 💡 **활용**: 생성된 키워드를 이 우선순위로 정렬하면 상위 노출 가능성이 높은 키워드를 먼저 볼 수 있음

### 3. 맛집 (restaurant)

**조합 공식**:
```
[지역] + [맛집/업종 카테고리] + [키워드(메뉴명)] + [수식어(선택)]
```

**신지도 기준**:
- 지역 + 맛집/업종 카테고리 (카페, 베이커리, 술집 등)
- 지역 + 메뉴명 (삼겹살, 파스타 등)
- 지역 + 메뉴명 + 맛집/식당/음식점

**필터 칩 키워드**:
- 업종: 한식, 일식, 중식, 양식, 카페, 고기요리, 디저트
- 편의시설: 영업중, 주차, 단체석, 예약가능, 포장가능
- 분위기: 데이트, 혼밥, 회식, 분위기좋은, 가성비

> ⚠️ "맛집 추천"과 "맛집"은 순위 로직상 동일 키워드로 취급될 수 있음

---

### 4. 미용실 (hairshop)

**특징**:
- 스타일 칩 자동 생성 (레이어드컷, 댄디컷, C컬펌 등)

**조합 예시**:
- `홍대 미용실` → `홍대 미용실 댄디컷`, `홍대 미용실 레이어드컷`

## 🔧 프로그램 기능 구상

### 핵심 기능
1. **키워드 대량 생성**: 신지도/구지도 키워드 자동 조합

### 기술 스택 (예정)
- Python + Selenium/Playwright: 웹 크롤링
- PyQt6 또는 Tkinter: GUI
- PyInstaller: EXE 패키징

### 업종별 추출 로직 요약

```python
# 맛집 (restaurant)
keywords = apollo_state["PlaceDetailBase:{ID}"]["keywords"]  # 직접 접근 OK

# 병의원 (hospital)

# Step 3: hospitalInfo에서 진료과목

```

### __ref 패턴 vs 직접 임베딩

Apollo State는 데이터를 **참조(`__ref`)** 또는 **직접 임베딩** 두 가지 방식으로 저장:

```javascript
// 참조 패턴 (__ref)
"informationTab": {"__ref": "InformationBase:12927872"}
// → apollo_state["InformationBase:12927872"]에서 실제 데이터 조회

// 직접 임베딩 패턴
"informationTab": {"__typename": "InformationBase", "keywordList": [...], ...}
// → 바로 keywordList 접근 가능
```

> 💡 **구현 팁**: 두 패턴 모두 처리하도록 분기 필요

### nxPlaces 쿼리 (순위 체크용)

**요청 형식**:
```graphql
query getAdditionalPlaces($input: PlacesInput) {
  businesses: nxPlaces(input: $input) {
    total
    items {
      id
      name
      businessCategory
      gdid
    }
  }

```

**Variables**:
```json
{
  "input": {
    "query": "강남맛집",
    "display": 30,
    "start": 1,

  }

```

| 파라미터 | 필수 | 설명 |
|:---|:---:|:---|
| `query` | ✅ | 검색 키워드 |

| `start` | ✅ | 시작 위치 (페이징) |
| `x`, `y` | ❌ | 좌표 (선택, 정확도 향상) |

### gdid 구조 분석

**형식**: `{ID},{CODE}:{SCORE1}:{SCORE2}:{SCORE3}`

**예시**:
```
1181608808,2:0.366856:0.569915:0.370457
│          │ │        │        │
│          │ │        │        └─ SCORE3 (위치/최신성 관련 추정)
│          │ │        └─ SCORE2 (인기도/품질 관련 추정)
│          │ └─ SCORE1 (검색어 관련성 - 동일 검색에서 거의 동일)
│          └─ CODE (알고리즘 코드, 현재 모두 "2")
└─ 플레이스 ID
```

**점수 합계와 순위 관계**:

| 업종 | SCORE1+SCORE2+SCORE3 | 순위 일치 여부 |
|:---|:---|:---:|
| 피부과 (hospital) | 1.518 → 1.494 → 1.488 → 1.474 → 1.466 | ✅ 완벽 |
| 꽃집 (place) | 1.160 → 1.150 → 1.129 → 1.118 → 1.111 | ✅ 완벽 |
| 미용실 (hairshop) | 1.332 → 1.321 → 1.316 → 1.314 → 1.311 | ✅ 완벽 |
| 맛집 (restaurant) | 1.307 → 1.292 → 1.297 → 1.290 → 1.291 | ⚠️ 2,3위 역전 |

> 💡 **결론**: 대부분의 업종에서 `SCORE1 + SCORE2 + SCORE3` 합계가 높을수록 상위 순위.
> 맛집의 경우 추가 요인(광고, 실시간 변동 등)이 영향을 미칠 수 있음.

**SCORE1 값 (검색어별)**:

| 검색어 | SCORE1 범위 | 비고 |
|:---|:---|:---|
| 강남맛집 | 0.366856 | 거의 동일 |
| 강남피부과 | 0.420277~0.420323 | 약간 변동 |
| 강남 꽃집 | 0.389904 | 거의 동일 |
| 강남 미용실 | 0.439343~0.439389 | 약간 변동 |

> 💡 SCORE1은 검색 쿼리와 결과의 관련성 점수로 추정. 동일 검색에서 거의 동일한 값.

### 지도 형태 판단 방법 (신지도 vs 구지도)

> 📅 2026-01-07 추가

**핵심 발견**: GraphQL API 응답만으로는 지도 형태를 판단할 수 없음!

#### 정확한 판단 방법

검색 HTML 페이지에서 `queryType` 추출:

```
URL: https://m.search.naver.com/search.naver?query={키워드}
정규식: /"queryType"\s*:\s*"([^"]+)"/
```

| queryType 값 | 지도 형태 |
|:---|:---|
| `restaurant` | 신지도 |
| `hospital` | 신지도 |
| `hairshop` | 신지도 |
| `cafe` | 신지도 |
| `nailshop` | 신지도 |
| `null` / 없음 | **구지도** |

#### "지도" 키워드 효과

| 키워드 | queryType | 지도 형태 |
|:---|:---|:---|
| 강남 맛집 | `restaurant` | 신지도 |
| 강남 맛집 지도 | `null` | **구지도** |
| 강남 피부과 | `hospital` | 신지도 |
| 강남 피부과 지도 | `null` | **구지도** |
| 강남 꽃집 | `null` | 구지도 |

> 💡 **결론**: 키워드에 "지도"를 추가하면 queryType이 null이 되어 구지도로 전환됨!

#### 구현 방식 (병렬 요청)

`rank_checker_graphql.py`에서 병렬 처리:

```python
# 두 요청을 동시에 시작
with ThreadPoolExecutor(max_workers=2) as executor:
    rank_future = executor.submit(self._fetch_rank_from_graphql, ...)  # 순위
    map_type_future = executor.submit(self._fetch_map_type, ...)       # 지도형태
```

**성능**: 5개 키워드 = 약 1.3초 (병렬 처리)
